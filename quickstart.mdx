This guide will walk you through creating a styled button using Centralised UI's ColorTint system. You'll go from installation to a fully working, reusable UI styling setup.

<Note>
Make sure you've completed the [installation](/installation) before starting this quickstart.
</Note>

## Create Your First Styled Button

<Steps>
  <Step title="Create a UI Button with CentralisedUIData">
    In your Unity scene Hierarchy, right-click and navigate to:
    
    **UI > ProgramadorCastellano > CentralisedUI > Button**
    
    This creates a Button GameObject with the `CentralisedUIData` component already attached. Alternatively, you can:
    
    1. Create a standard button: **UI > Button - TextMeshPro**
    2. Select the Button GameObject
    3. Click **Add Component** and search for "CentralisedUIData"
    4. Add the `CentralisedUIData` component (from the `ProgramadorCastellano.CentralisedUI` namespace)
    
    <Info>
    The `CentralisedUIData` component requires a `Selectable` component, which buttons already have.
    </Info>
  </Step>

  <Step title="Create a ColorTint ScriptableObject">
    Now create a reusable ColorTint styling asset:
    
    1. In your Project window, navigate to a folder where you want to store UI styling assets (e.g., `Assets/UI/Styles`)
    2. Right-click in the folder and select:
       
       **Create > ProgramadorCastellano > CentralisedUI > ColorTint**
    
    3. Name the new ScriptableObject something descriptive like "PrimaryButtonColors" or "DefaultColorTint"
    
    <Tip>
    Organize your UI styling ScriptableObjects in a dedicated folder like `Assets/UI/Styles` for easy reuse across your project.
    </Tip>
  </Step>

  <Step title="Configure the ColorBlock values">
    Select the ColorTint ScriptableObject you just created and configure its ColorBlock in the Inspector:
    
    The `Colors` field is a standard Unity `ColorBlock` with the following properties:
    
    - **Normal Color**: The default color (e.g., white or your brand color)
    - **Highlighted Color**: Color when hovering (e.g., slightly brighter)
    - **Pressed Color**: Color when clicking (e.g., darker)
    - **Selected Color**: Color when selected (e.g., your accent color)
    - **Disabled Color**: Color when disabled (e.g., gray)
    - **Color Multiplier**: Intensity multiplier (default is 1)
    - **Fade Duration**: Transition speed between states (default is 0.1)
    
    Try these example values for a blue primary button:
    
    ```text
    Normal Color: RGB(41, 128, 185) - Blue
    Highlighted Color: RGB(52, 152, 219) - Lighter Blue
    Pressed Color: RGB(21, 67, 96) - Dark Blue
    Selected Color: RGB(52, 152, 219) - Lighter Blue
    Disabled Color: RGB(127, 140, 141) - Gray
    Color Multiplier: 1
    Fade Duration: 0.1
    ```
    
    <Info>
    These are the same ColorBlock settings you'd normally configure directly on a Selectable component, but now they're reusable across multiple UI elements.
    </Info>
  </Step>

  <Step title="Assign ColorTint to CentralisedUIData and apply">
    Return to your Button GameObject in the Hierarchy and select it:
    
    1. In the Inspector, find the `CentralisedUIData` component
    2. Set **Transition Mode** to `ColorTint` (if not already selected)
    3. In the **Color Tint SO** field under the "Color Tint" header, drag and drop your ColorTint ScriptableObject
    4. Click the **Apply Skin** button in the Inspector
    
    You should immediately see your button update with the colors defined in your ScriptableObject. The underlying `Button` component's Colors property will be updated to match.
    
    <Tip>
    The "Apply Skin" button lets you preview changes in the Editor. At runtime, the styling is automatically applied in the `Awake()` method.
    </Tip>
  </Step>

  <Step title="Test in Play mode">
    Press Play in the Unity Editor and interact with your button:
    
    - **Hover**: The button should change to the Highlighted Color
    - **Click**: The button should change to the Pressed Color
    - **Release**: The button should return to the Normal Color
    
    The transitions should be smooth, with the fade duration you configured (default 0.1 seconds).
    
    <Info>
    When entering Play mode, the `CentralisedUIData` component's `Awake()` method automatically calls `OnSkinUI()`, which applies your styling configuration to the Selectable.
    </Info>
  </Step>
</Steps>

## Reuse Your Styling

Now that you have a ColorTint ScriptableObject, you can reuse it across multiple buttons:

<Steps>
  <Step title="Create additional buttons">
    Create more buttons in your scene using the same method as Step 1.
  </Step>

  <Step title="Apply the same ColorTint">
    For each new button:
    1. Select the button GameObject
    2. Find the `CentralisedUIData` component
    3. Assign the same ColorTint ScriptableObject to the **Color Tint SO** field
    4. Click **Apply Skin**
  </Step>

  <Step title="Update all buttons at once">
    To change the color scheme for all buttons:
    
    1. Select your ColorTint ScriptableObject in the Project window
    2. Modify the ColorBlock values in the Inspector
    3. In the scene, click **Apply Skin** on each button, or simply enter Play mode
    
    All buttons using this ScriptableObject will automatically use the updated colors at runtime.
  </Step>
</Steps>

<Tip>
This is the power of Centralised UI - change your ColorTint ScriptableObject once, and all UI elements using it are updated automatically at runtime. No need to update each button individually!
</Tip>

## Try Other Transition Types

Centralised UI supports all Unity Selectable transition modes:

### SpriteSwap Transition

For image-based buttons that swap sprites on interaction:

1. Create a SpriteSwap ScriptableObject: **Create > ProgramadorCastellano > CentralisedUI > SpriteSwap**
2. Configure the `Sprites` field with different sprite references for Highlighted, Pressed, Selected, and Disabled states
3. In `CentralisedUIData`, set **Transition Mode** to `SpriteSwap`
4. Assign the SpriteSwap ScriptableObject to the **Sprite Swap SO** field
5. Click **Apply Skin**

### Animation Transition

For buttons with animator-driven transitions:

1. Create an Animation ScriptableObject: **Create > ProgramadorCastellano > CentralisedUI > Animation**
2. Configure the `Animations` field with AnimationTrigger names for Normal, Highlighted, Pressed, Selected, and Disabled states
3. Ensure your button has an Animator component with matching triggers
4. In `CentralisedUIData`, set **Transition Mode** to `Animation`
5. Assign the Animation ScriptableObject to the **Animation SO** field
6. Click **Apply Skin**

<Warning>
For Animation transitions, you must have an Animator component on your UI element with animation triggers that match the names in your AnimationTriggers configuration.
</Warning>

## Understanding the Code

Here's what happens behind the scenes when you click "Apply Skin" or enter Play mode:

```csharp
// From CentralisedUIData.cs
namespace ProgramadorCastellano.CentralisedUI
{
    public class CentralisedUIData : MonoBehaviour
    {
        // The component automatically applies styling in Awake()
        private void Awake()
        {
            Initialize();
            OnSkinUI();
        }
        
        // OnSkinUI() updates the Selectable based on transition mode
        public void OnSkinUI()
        {
            _selectable.transition = _transitionMode;
            
            // For ColorTint mode:
            // _selectable.colors = _colorTintSO.Colors;
            
            // For SpriteSwap mode:
            // _selectable.spriteState = _spriteSwapSO.Sprites;
            
            // For Animation mode:
            // _selectable.animationTriggers = _animationSO.Animations;
        }
    }
}
```

The ColorTint ScriptableObject is simple:

```csharp
// From CentralisedUIColorTintSO.cs
namespace ProgramadorCastellano.CentralisedUI
{
    [CreateAssetMenu(menuName = "ProgramadorCastellano/CentralisedUI/ColorTint", order = 1)]
    public class CentralisedUIColorTintSO : ScriptableObject
    {
        [SerializeField] ColorBlock _colors = ColorBlock.defaultColorBlock;
        public ColorBlock Colors => _colors;
    }
}
```
